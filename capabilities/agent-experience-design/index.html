Capability / Autonomous Workflows

Agent Experience Design

Making multi-step agent behavior visible, steerable, and trustworthy through plan design, progress feedback, and intervention controls.

Plan visibility Execution feedback Handoff and control

Focus Areas

Designing agent flows that expose intent, intermediate steps, and checkpoints rather than hiding automation behind a black box.

  • Legible plans and step states
  • Progress models for long-running tasks
  • Pause, edit, redirect, and stop controls
  • Balanced autonomy with meaningful oversight

As AI systems evolve, they are increasingly able to perform tasks rather than simply generate information.

These systems are often referred to as AI agents. Instead of responding to a single request, an agent can plan actions, interact with external tools, gather information, and execute multi-step workflows.

Designing experiences for these systems requires a new way of thinking about interfaces.

From Responses to Actions

Traditional AI interfaces focus on responses. A user asks a question, and the system produces an answer.

Agent systems operate differently. They perform actions on behalf of the user. An agent might search multiple sources, retrieve documents, analyze information, generate a report, and update a workspace within a single workflow.

The role of design is to make these actions visible and understandable.

Making Agent Behavior Legible

One of the biggest challenges with autonomous systems is opacity. If an agent performs multiple operations behind the scenes, users may struggle to understand what is happening and trust erodes quickly.

Good agent design exposes the structure of the task.

  • the goal of the task
  • the plan the agent intends to follow
  • which steps are currently being executed
  • what results have been produced so far

Progress and Feedback

Tasks performed by agents often take time. Unlike instant responses from a model, multi-step workflows may involve several stages.

Users should see the system moving forward, completing steps, and gradually building toward the final outcome. Progress indicators, step visualization, and status updates help maintain clarity throughout the process.

Designing for Intervention

Even highly capable agents will occasionally misunderstand instructions or pursue the wrong strategy.

Users must always have the ability to intervene. This might include editing a step in the plan, redirecting the search strategy, modifying the goal, or stopping execution entirely.

Agent design should feel less like pressing a button and more like directing an assistant.

The Role of Autonomy

Too little autonomy and the system becomes tedious to operate. Too much autonomy and users lose confidence in the outcome.

Designing agent experiences involves finding the right level of independence for the system while keeping humans meaningfully involved in the process. When this balance is achieved, agents can dramatically simplify complex workflows.
